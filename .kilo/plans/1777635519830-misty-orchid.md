# KALLAA E-commerce Fixes Plan

## Overview
Fix currency inconsistency (ensure all pricing uses INR ₹) and mobile responsiveness issues for product displays. Backend stores prices correctly as numbers, so focus on frontend refactoring.

## Issue 1: Currency Consistency (INR ₹)

### Steps:
1. **Verify/Expand formatPrice Utility**
   - Confirm `frontend/src/utils/formatPrice.js` exists with proper INR formatting
   - Ensure it handles edge cases (null/undefined prices)

2. **Refactor AdminDashboard Component**
   - Replace `$` symbols with `formatPrice()` calls
   - Update column headers from "Price ($)" to "Price (₹)"
   - Use dynamic `product.price` instead of hardcoded values

3. **Refactor Checkout Component**
   - Replace `$$` with `formatPrice()` for totals and subtotals
   - Ensure all price calculations use dynamic product prices

4. **Refactor Cart Component**
   - Replace `$` symbols with `formatPrice()` calls
   - Verify price display logic uses product.price

5. **Audit All Product Components**
   - Check ProductCard, ProductDetail, Gallery, Products pages
   - Replace any hardcoded prices (like 3000) with dynamic product.price
   - Ensure all use formatPrice() utility

6. **Backend Verification**
   - Confirm Product model uses Number type for price (already correct)
   - Ensure API responses send numeric prices (already correct)

## Issue 2: Mobile Responsiveness

### Steps:
1. **Fix Product Grid Layouts**
   - Update Products.tsx grid to: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6`
   - Update ProductDetail.tsx for better tablet layout
   - Update ProductInfo.tsx if needed

2. **Fix Container Widths**
   - Replace any fixed widths (if found) with `w-full max-w-7xl mx-auto px-4`
   - Ensure responsive padding and margins

3. **Improve Card Responsiveness**
   - Ensure ProductCard uses `className="w-full"`
   - Update image classes to `w-full h-auto object-cover`
   - Verify aspect ratios work well on mobile

4. **Remove Mobile Visibility Issues**
   - Check for and remove any `hidden md:block` or similar
   - Fix any `overflow-hidden` that cuts content
   - Ensure proper breakpoints for text sizing (add `sm:` where missing)

5. **Add Product Fallback**
   - Add no-products message in Products.tsx: `{products.length === 0 && <p className="text-center text-gray-400">No pieces available</p>}`

## Bonus UX Improvements
1. **Price Formatting Enhancement**
   - Verify formatPrice uses proper Indian number formatting (₹3,000)

2. **Hover Effects**
   - Add subtle scale and transition to product cards: `hover:scale-105 transition-transform duration-200`

3. **Consistent Spacing**
   - Review and standardize gaps/padding across components for mobile

## Validation Steps
- Test on mobile devices/simulator
- Verify all prices show ₹ consistently
- Check API calls remain unchanged
- Run existing tests if available

## No Changes Needed
- Backend API logic (already sends numeric prices)
- Business logic (product availability, cart calculations)
- Database schema (already uses Number/decimal)

This plan addresses all requirements while maintaining existing functionality.
